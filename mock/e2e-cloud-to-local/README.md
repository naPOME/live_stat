# E2E Cloud -> Local Mock

This folder is generated by `apps/cloud/scripts/e2e_seed_export.mjs`.
It contains a full mock export generated from real Supabase data so you can test the entire flow.

## Generate

From repo root:

```powershell
node apps\cloud\scripts\e2e_seed_export.mjs
```

If your Cloud app runs on a different URL, set it first:

```powershell
$env:CLOUD_APP_ORIGIN = "http://localhost:3002"
```

## Use in Local App

1. Copy logos into `C:\logo` (or run the helper):

```powershell
.\mock\e2e-cloud-to-local\install.ps1
```

2. Point the local app at the roster mapping:

```powershell
$env:ROSTER_MAPPING_PATH = "${PWD}\mock\e2e-cloud-to-local\roster_mapping.json"
```

3. Start the local app and verify:
- Team names + colors in overlays
- Slot mapping correctness
- Player stats in results/fraggers

## Public Standings Check

Open:

```
/standings/<tournament_id>
```

You can grab `tournament_id` from `mock/e2e-cloud-to-local/seed.json`.

## Cleanup

When done, remove this mock data from Supabase:

```powershell
node apps\cloud\scripts\e2e_cleanup.mjs
```

You can delete this folder afterwards.
